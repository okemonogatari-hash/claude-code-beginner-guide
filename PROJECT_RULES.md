# 共同編集・正本管理ルール

最終更新: 2026-05-07

このファイルは、Claude Code 初心者向け虎の巻を複数人と複数AIで安全に育てるためのルールです。

## 1. 正本の決め方

- 正本は GitHub リポジトリ `okemonogatari-hash/claude-code-beginner-guide` の `main` ブランチです
- 公開物は GitHub Pages の `https://okemonogatari-hash.github.io/claude-code-beginner-guide/` です
- 森側のノートは、進行管理、素材メモ、意思決定ログです
- 古い `html-share-box` 系URLは正本ではありません

## 2. 変更してよい範囲

| 種別 | 方針 |
|---|---|
| 誤字、表記ゆれ、リンク切れ | そのまま修正してOK |
| FAQ、短い補足、カード追加 | 追加OK。参照元がある場合は `SOURCES.md` も更新 |
| 章構成、デザイン方針、大きな削除 | 事前にチャットで一声かける |
| 公開URL、ファイル名、リポジトリ設定 | あおいCDO、オーナー確認後に変更 |
| 会員向け記事やチャット内容 | 全文転載しない。要約、短い引用、リンクで扱う |

## 3. 共同編集の基本

- 1コミット1目的を目安にする
- コミットメッセージは日本語でOK
- 他の人やAIが触った変更を勝手に戻さない
- 大きな方針変更をするときは、理由と影響範囲を残す
- 共同編集者の視点を尊重し、初心者目線を最優先にする

## 4. AIが触るときの手順

AIや別チャットがこのリポジトリを触るときは、必ずこの順で確認します。

1. `README.md` を読む
2. この `PROJECT_RULES.md` を読む
3. `SOURCES.md` を読む
4. `git status` またはGitHub上の最新状態を確認する
5. 編集対象のファイルだけを変更する
6. 変更したリンクや参照元があれば `SOURCES.md` に追記する
7. 公開URLで表示とリンクを確認する
8. 森側の進捗管理メモに更新内容を残す

## 5. 素材と著作権の扱い

- リベシティ内記事、学長マガジン、チャット本文は会員向け情報として扱う
- 公開ページには、元記事の全文や長い引用を載せない
- 必要な場合は、要点、構造、短い引用、リンクで残す
- 著者名、記事ID、source URLがわかるものは `SOURCES.md` に残す
- 不明な素材は、公開前に出典を確認する

## 6. 個人情報と安全

公開ページやGitHubには以下を入れません。

- APIキー、トークン、ログイン情報
- 会員の個人情報、個別の相談内容、許可のない発言
- ローカルPCの秘密情報が入ったパス
- 未確認の料金、仕様、提供状況を断定する記述

## 7. リリース前チェック

変更後は最低限これを確認します。

- 本編または別冊ページが開ける
- 追加したリンクが開ける
- スマホ幅で文字がはみ出していない
- `README.md` の公開ページ一覧と実ファイルが一致している
- 参照元を追加した場合、`SOURCES.md` に残っている

## 8. 迷ったときの判断基準

この虎の巻は「詳しい人が満足する資料」よりも、「怖がっている初心者が最初の一歩を踏める資料」を優先します。

専門用語を増やすより、手順を短くする。機能を盛るより、不安を減らす。これを判断の中心に置きます。
