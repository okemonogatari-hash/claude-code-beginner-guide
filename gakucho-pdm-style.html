Claude Codeを
会社化する ── 学長スタイル

両学長が4/30ライブで語った「Claude Code＝会社化」の発想を、
初心者でもそのままコピペで使えるプロンプト5パターンに落とし込んだ別冊やで。
本編の「ちょっと先まで行きたい人」向けの、もう一段奥の話や〜。

Section 01

学長が言うてた、「Claude Code＝スーパー会社」の話

学長が4/30ライブで「Claude Code は会社をイメージしたらええ」って言うてたんよ。
ちょっと長いけどな、これがめちゃ刺さるから引用するで〜。

両学長 / リベ大ライブ 2026-04-30

「クロちゃんに頼むと、このクロちゃんが全部の責任者なわけですよ。 ま、クロちゃんが今後全部の入り口になるんですよ。

スーパー会社をイメージしてください、皆さん。 会社をイメージしてスーパー秘書みたいな秘書でもいいし、総務でもいいですわ。

とにかくこのクロちゃんにお願いしたら、このクロちゃん経由で 経理部にも連絡してくれる、営業部にも連絡してくれる、製品開発部にも連絡してくれる みたいな、仲介役みたいな。」

学長の体制（PDM 1人） → 意味
クロちゃん（PDM） ＝ 全部の入り口・全部の責任者・スーパー秘書
経理部 ← クロちゃん経由で連絡（数字の処理）
営業部 ← クロちゃん経由で連絡（外への動き）
製品開発部 ← クロちゃん経由で連絡（モノづくり）

要するにな、Claude Code は 「責任者を1人立てて、その子経由で全部やる」 っていう体制で動かすんがコツや、っていう話やねん。
これな、初心者にとってもめちゃ大事や。「色んなAIをバラバラに使う」じゃなくて、「ひとつ責任者を立てる」。これだけで一気に楽になるで〜。

Section 02

そのまま使える 学長スタイル指示プロンプト 5パターン

学長の話し方をベースにした、コピペで使える指示文5つやで。
最初は ① から順に試してみて〜。「最初の声かけ」が一番効くで。

01

PDM（責任者）設定の最初の声かけ

使うとき：Claude Codeを起動して、最初の1発目に投げる声かけや。これで「お互いの立ち位置」が決まるで。

今日からあんたは僕の専属PDM（プロダクトマネージャー）やで。 僕からの指示は全部あんたに投げるから、必要に応じて 関連の部署（経理・営業・開発）に振り分けて動いてな。 分からんことあったら遠慮なく聞いて。
「あんたが責任者な」って最初に決めるだけで、その後の動きが全然変わるで。
02

セッションが重くなった時の引き継ぎ依頼

使うとき：会話が長くなって、Claudeの動きがモッサリしてきた時。新しいセッションへ「申し送り書」を作ってもらう。

最近重なってきたから、新しいPDMさんに引き継ぎたい。 今までのやり取り全部まとめて、引き継ぎ書作って。 ・どんな話してたか ・何が決まってるか ・次にやることリスト これ全部入った申し送り書、頼むわ。
学長いわく「ニューPDMじゃ〜ん爆弾」や。新しいセッション開いて、この引き継ぎ書ペッと貼ったら復活するで。
03

ツール乗り換え（他のAIから Claude Code へ）

使うとき：今まで ChatGPT や Claude Cowork で進めてた案件を、Claude Code に引っ越す時。学長の表現でいうと「部署異動の申し送り書」や。

これからCodeに移りたいから、今まで使ってたあんたから Codeへの引き継ぎ書を作って。 僕の癖・好み・進行中の案件、全部書いといてな。 ペッと貼り付けて新しい子に渡すから、 それで困らんようにまとめてくれ。
「総務部から経理部へ異動」みたいな感覚や。引き継ぎ書ができたら、Claude Code 側にコピペするだけ。
04

部下（部署）への仲介依頼

使うとき：1つの作業の中で、複数の専門領域（数字・コード・文章）が絡んでくる時。PDMに「適切な部署に振って」とお願いする。

この作業、いくつかの担当に振り分けて進めてほしい。 ・数字まわり → 経理部担当の子に ・コードまわり → 開発部担当の子に ・文章まわり → 広報部担当の子に PDMのあんたが全体まとめて、最後に僕に報告してな。
最初は「部署イメージ」だけでOK。慣れてきたら「経理部のしーちゃん」みたいに名前付けると、もっと指示が通りやすくなるで〜。
05

エラー対応（スクショ貼り付けスタイル）

使うとき：何かしてて画面にエラーが出た時。スクショを貼り付けて、PDMに「対応してくれ」と投げる。

[ここにスクショを貼り付け] こういうエラー画面が出とるんやけど、 何をどう操作したら解決する？ 代わりにやってくれてもええで。
エラー画面は文字で説明するより、スクショ1発が早いで。Claude Code はちゃんと読んでくれる。
Section 03

「ニューPDMじゃ〜ん爆弾」の使い方

Claude Code は使い続けるとセッションが重くなる。学長の解決策がこれ ──「引き継ぎ書を作って、新しいセッションにペッと貼る」だけや。

🦁 学長の運用（4/30ライブから）

「2週間に1回ぐらいかな、重たくなってくるんで、新しいセッションに変えたりはしてるんですけどね。
それも引き継いでと、この PDMからニューPDMに引き継いでって、引き継ぎ書作ってもらったらペッと書類作ってくれるから、
新しいセッション作って、またその時ニューセッションで立ち上げて、
過去前の今まで使ってたPDMさんからの引き継ぎ書をここにポイっと入れて、
そしたら新しい ニューPDMじゃ〜ん爆弾 みたいな感じでね。動いてくれる。快適に動いてくれる。」

使い方は3ステップだけや：

  1. 今のClaudeに「引き継ぎ書 作って」って頼む（プロンプト ② 参照）
  2. 新しいセッションを立ち上げる（コマンド claude でもう一回起動するだけ）
  3. 新しいClaudeに、さっきの引き継ぎ書を ペッと貼り付ける

※ Claude Code の場合、/compact コマンドでも似たことができるで。「全部リセットせず、要点だけ残して続ける」やつ。本編 Chapter 7 で紹介済みや。

🌀 大事なのは 「忘れる前提で仕組み化」

AIも自分も、どっちも「ぜんぶ覚えとく」のは無理。
だから 引き継ぎ書を残す習慣 を持っとくと、何回でもやり直せる。
これがあるだけで、AIとの付き合いが めちゃ気楽になる で〜。

「自分専用の小さな会社」を、
あんたも今日から立ち上げてみい〜🦁

最初は 01 の「PDM設定」を1回投げるだけでええ。
それだけで Claude Code との距離感が、ガラっと変わるで。
